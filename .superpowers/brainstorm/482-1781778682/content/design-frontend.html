Design: Vue Pages & Components

What the user sees — routes, views, and which components they use

Public Routes (no login needed)

/ HomeView.vue

Hero banner · Featured cars carousel · New arrivals · Deals of the day

Components: Navbar · CarCard · HeroBanner · Footer

/cars CarsView.vue

Full car listing with sidebar filters + pagination

Components: CarFilter · CarCard · Pagination

/cars/:id CarDetailView.vue

Car photos · specs · Buy / Lease buttons (login required to proceed)

Components: CarGallery · OrderModal

/login   /register LoginView.vue

JWT login · client registration · stores token in Pinia auth store

Protected Routes (JWT required — redirect to /login)

/dashboard Client

My orders · profile details · order status tracking

Components: OrderList · ProfileForm

/admin/cars Employee+

Car listing management — add, edit, delete with image upload

Components: CarTable · CarFormModal

/admin/orders Employee+

All orders · approve / deny / mark complete

Components: OrderTable · StatusBadge

/admin/users Admin only

Manage users · assign roles · deactivate accounts

Components: UserTable · RoleBadge

Pinia Stores (state management)
auth.js
  • token (persisted to localStorage)
  • user { id, name, email, role }
  • actions: login(), logout(), register()
  • getter: isLoggedIn, isAdmin, isEmployee
cars.js
  • cars[], filters{}, pagination{}
  • actions: fetchCars(), fetchFilters()
  • Router guard checks auth store on every /admin/* route
