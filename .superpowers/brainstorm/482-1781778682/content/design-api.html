Design: REST API Endpoints

Clean naming, proper verbs, pagination on lists, error messages everywhere

Auth (public — no JWT required)

MethodEndpointDescriptionReturns
POST/api/auth/loginLogin with email + password{ token, user }
POST/api/auth/registerRegister new client{ token, user }

Cars (GET public · POST/PUT/DELETE require employee+ role)

MethodEndpointDescriptionNotes
GET/api/carsList all cars?page=1&limit=12&brand=BMW&min_price=5000
GET/api/cars/{id}Single car detail404 if not found
POST/api/carsAdd new car listingReturns new car with id
PUT/api/cars/{id}Update car detailsPartial update supported
DELETE/api/cars/{id}Remove car listingAdmin only
GET/api/cars/filtersFilter options (brands, years…)Used to populate filter UI

Orders (all require JWT)

MethodEndpointDescriptionNotes
GET/api/ordersList ordersClient sees own · Employee sees all · ?page=1
GET/api/orders/{id}Single order403 if not owner/employee
POST/api/ordersCreate order (buy/lease)Returns new order with id
PUT/api/orders/{id}Update order statusEmployee+ only

Users (admin only except own profile)

MethodEndpointDescriptionNotes
GET/api/usersList all usersAdmin only · ?page=1
GET/api/users/{id}User profileOwn profile or admin
PUT/api/users/{id}Update user / change roleRole change = admin only
DELETE/api/users/{id}Remove userAdmin only
Pagination response shape (all list endpoints)
